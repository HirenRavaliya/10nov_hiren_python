Virtual Environment &
Django Installation

Setting up an isolated Python environment and installing Django the right way.

VE
What is a Virtual Environment?

A virtual environment is an isolated Python workspace that keeps your project's dependencies completely separate from the global Python installation and from other projects on the same machine.

Without it, every package you install goes into the global Python directory — this quickly leads to version conflicts when two projects need different versions of the same library.

Think of it like this: Each project gets its own room with its own bookshelf. Books (packages) in one room don't interfere with books in another.
IS
Isolation
Each project has its own set of packages with exact versions.
RP
Reproducibility
Share requirements.txt so anyone can recreate the same setup.
CS
Clean System
Global Python stays uncluttered and stable.
NC
No Conflicts
Project A using Django 4.x and Project B using Django 3.x — no problem.
SS
Step-by-Step Setup Process
STEP 01 Check Python Installation

Ensure Python 3 is installed on your system before proceeding.

Terminal
python --version # Windows python3 --version # Linux / macOS
If Python is not installed, download it from python.org and make sure to check "Add Python to PATH" during installation.
STEP 02 Create a Project Folder

Create a dedicated directory for your Django project.

Terminal
mkdir myproject cd myproject
STEP 03 Create the Virtual Environment

Use Python's built-in venv module to create an isolated environment inside your project folder.

Terminal
# Creates a folder named 'venv' with isolated Python python -m venv venv # Windows python3 -m venv venv # Linux / macOS

A folder named venv/ is created containing a private Python binary and pip.

STEP 04 Activate the Virtual Environment

Activating tells your terminal to use the isolated Python instead of the global one.

Terminal
# Windows (Command Prompt) venv\Scripts\activate # Windows (PowerShell) venv\Scripts\Activate.ps1 # Linux / macOS source venv/bin/activate
After activation, your terminal prompt will show (venv) at the start — this confirms the environment is active.
STEP 05 Upgrade pip

It's good practice to upgrade pip inside the virtual environment before installing packages.

Terminal
pip install --upgrade pip
STEP 06 Install Django

Install Django using pip. Since the virtual environment is active, Django will be installed only inside it.

Terminal
pip install django # Or install a specific version: pip install django==4.2
STEP 07 Verify Django Installation

Confirm that Django was installed correctly.

Terminal
python -m django --version # Output: 4.2.x (or whatever version was installed)
STEP 08 Create a Django Project

Start a new Django project using the django-admin command.

Terminal
django-admin startproject mysite . # The dot (.) creates the project in the current directory
STEP 09 Save Dependencies

Export all installed packages to a file so the environment can be recreated later.

Terminal
pip freeze > requirements.txt
STEP 10 Deactivate When Done

To exit the virtual environment when you're finished working.

Terminal
deactivate
