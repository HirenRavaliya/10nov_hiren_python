What is GitHub & Version Control?

Version Control is a system that records changes to files over time so you can recall specific versions later, collaborate with others, and track who changed what and when.

Git is the most widely used version control system. GitHub is a cloud-based hosting platform for Git repositories — it lets you store your code online, collaborate, and manage project history through a web interface.

Why GitHub for Django projects? It keeps your code safe in the cloud, lets teammates collaborate without overwriting each other's work, and acts as a portfolio for your projects.
RE
Repository (Repo)
A folder tracked by Git — contains all your project files and the full history of changes.
CM
Commit
A snapshot of your project at a given point in time. Each commit has a unique ID and message.
BR
Branch
A parallel version of your project. Work on features without touching the main stable code.
PP
Push / Pull
Push = upload local commits to GitHub. Pull = download remote changes to your local machine.

Step-by-Step: Upload Django Project to GitHub
STEP 01 Install Git

Ensure Git is installed on your machine.

Terminal
git --version # If not installed: https://git-scm.com/downloads
STEP 02 Configure Git Identity

Set your name and email — this info is attached to every commit you make.

Terminal
git config --global user.name "Your Name" git config --global user.email "you@example.com"
STEP 03 Create a .gitignore File

Before initializing Git, create a .gitignore file to exclude files that should NOT be tracked (virtual environment, database, secrets, etc.).

.gitignore
# Virtual Environment venv/ env/ # Python cache __pycache__/ *.pyc *.pyo # Django *.sqlite3 media/ staticfiles/ # Environment variables / secrets .env # IDE files .vscode/ .idea/
Never upload your virtual environment folder or .env secrets to GitHub — this is a critical security and size consideration.
STEP 04 Initialize Git in Project Folder

Navigate to your Django project folder and initialize a Git repository.

Terminal
cd myproject # go to your project folder git init # initializes an empty git repo
STEP 05 Stage All Project Files

Add all files to the staging area (files listed in .gitignore will be excluded automatically).

Terminal
git add . # stages all files for the first commit git status # verify what will be committed
STEP 06 Create the First Commit

Commit the staged files with a descriptive message.

Terminal
git commit -m "Initial commit: Django project setup"
STEP 07 Create a Repository on GitHub

Go to github.com, click New Repository, give it a name (e.g., django-project), set it to Public or Private, and click Create repository. Do not initialize with a README (since you already have local files).

STEP 08 Link Local Repo to GitHub Remote

Connect your local repository to the GitHub repository you just created.

Terminal
git remote add origin https://github.com/username/django-project.git # Replace 'username' and 'django-project' with your actual values
STEP 09 Rename Branch to 'main' (Optional but Recommended)

GitHub defaults to 'main' as the primary branch name.

Terminal
git branch -M main
STEP 10 Push Project to GitHub

Upload your committed code to GitHub. The -u flag sets the upstream, so future pushes only need git push.

Terminal
git push -u origin main # Enter your GitHub username and personal access token when prompted
GitHub no longer accepts passwords — use a Personal Access Token (PAT). Generate one at: GitHub → Settings → Developer Settings → Personal Access Tokens.
STEP 11 Subsequent Updates

After making changes to your Django project, use these three commands to push updates to GitHub.

Terminal
git add . # stage changes git commit -m "feat: added user login view" # commit git push # push to GitHub
