How Google Maps API Works in This App

  1. 1 Settings: GOOGLE_MAPS_API_KEY is stored in settings.py and passed to templates via the view's context dictionary.
  2. 2 Data serialisation: Django's template engine serialises the DOCTORS list (including lat/lng) into a JavaScript array using {% verbatim %}{% for doc in doctors %}{% endverbatim %} loops.
  3. 3 Async load: The Maps JS API is loaded with callback=initMap&loading=async — the browser fetches the script without blocking the page.
  4. 4 Map init: initMap() creates a google.maps.Map centred on Gujarat, applies a monochrome style, then loops DOCTORS to place a google.maps.Marker at each clinic's coordinates.
  5. 5 Info Windows: Each marker gets a google.maps.InfoWindow containing the doctor's name, address, timings, phone, and action buttons — opening on marker click.
  6. 6 Specialty filter: Sidebar chips call marker.setVisible(true/false) to show/hide markers without reloading the page, keeping the map interactive.
  7. 7 Detail page mini-map: doctor_detail view passes a single doctor's lat/lng to a second smaller map instance, demonstrating reuse of the same API across multiple URL routes.
